# Node Schema Change Policy

**Status:** `[x]`  
**Epic:** G — Governance and Policies  
**Owner:** Architecture team

---

## 1. Requirements

1. Schema changes require explicit version bumps.
2. Provide forward and backward compatibility notes.
3. Include migration scripts or adapters when breaking changes occur.
4. Update schema diffs and change rationale in the ActionLog.
